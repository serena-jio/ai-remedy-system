# AI客服智能补救系统 - 部署指南

## 方案一：Cloudflare Tunnel（推荐 ⭐⭐⭐⭐⭐）

### 优点
- ✅ 完全免费
- ✅ 无需公网IP
- ✅ 无需配置路由器
- ✅ 自动HTTPS加密
- ✅ 访问速度快（Cloudflare全球CDN）
- ✅ 安全可靠

### 步骤

#### 1. 安装 Cloudflare Tunnel
```bash
# macOS 安装
brew install cloudflared

# 或直接下载
# https://github.com/cloudflare/cloudflared/releases
```

#### 2. 登录 Cloudflare
```bash
cloudflared tunnel login
```
这会打开浏览器，用您的Cloudflare账号登录（没有账号的话免费注册一个）

#### 3. 创建隧道
```bash
cd /Users/jinghongjiao/CascadeProjects/splitwise/ai-remedy-system
cloudflared tunnel create ai-remedy-system
```

#### 4. 启动隧道
```bash
cloudflared tunnel --url http://localhost:8000
```

系统会给您一个公网地址，例如：
```
https://random-name-123.trycloudflare.com
```

**将这个地址分享给任何人，他们都可以直接访问！**

## 方案二：局域网共享（同一WiFi下）

### 适用场景
- 只需要同一办公室/家里的人访问
- 演示或内部使用

### 步骤

#### 1. 查看您的本机IP地址
```bash
# macOS/Linux
ifconfig | grep "inet " | grep -v 127.0.0.1

# 或者
ipconfig getifaddr en0
```

假设您的IP是：`192.168.1.100`

#### 2. 确保服务器运行
```bash
cd /Users/jinghongjiao/CascadeProjects/splitwise/ai-remedy-system
python3 -m http.server 8000
```

#### 3. 分享访问地址
其他人在同一WiFi下访问：
```
http://192.168.1.100:8000
```

### 注意事项
- ⚠️ 需要关闭防火墙或允许8000端口
- ⚠️ 只能在同一局域网内访问
- ⚠️ IP地址可能会变化

## 方案三：GitHub Pages 部署（永久免费）

### 优点
- ✅ 完全免费
- ✅ 永久在线
- ✅ 自动HTTPS
- ✅ 全球访问
- ✅ 无需维护服务器

### 步骤

#### 1. 创建 GitHub 仓库
```bash
cd /Users/jinghongjiao/CascadeProjects/splitwise/ai-remedy-system

# 初始化 git
git init
git add .
git commit -m "Initial commit: AI客服智能补救系统"

# 创建 GitHub 仓库后
git remote add origin https://github.com/你的用户名/ai-remedy-system.git
git branch -M main
git push -u origin main
```

#### 2. 启用 GitHub Pages
1. 进入仓库设置（Settings）
2. 找到 Pages 选项
3. Source 选择 `main` 分支
4. 点击 Save

#### 3. 访问地址
几分钟后，您的网站会发布在：
```
https://你的用户名.github.io/ai-remedy-system/
```

## 方案五：使用 Serveo（临时分享）

### 优点
- ✅ 无需安装
- ✅ 一条命令搞定
- ✅ 适合临时演示

### 步骤

```bash
# 确保服务器在8000端口运行
# 然后在新终端执行：
ssh -R 80:localhost:8000 serveo.net
```

会得到一个临时地址：
```
https://random.serveo.net
```

### 注意
- ⚠️ 地址是临时的，关闭终端后失效
- ⚠️ 适合短期演示
